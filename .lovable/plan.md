
目标改动分为 3 组，统一处理为“顶栏简化 + 侧栏可折叠修复 + 市场看板改成单主工作区”。

1. 顶栏重排
- 删除顶栏中的闪电图标和“电力交易管理系统”文字。
- 把原来单独一行的 P1 消息并入顶栏这一行展示，保留：
  - P1 标识
  - 时间
  - 消息标题/摘要
  - 轮播指示
  - 关闭 X 按键
- 顶栏右侧继续保留通知铃铛和用户头像。
- 对应实现会调整：
  - `src/components/layout/AppTopBar.tsx`：改成“左侧折叠按钮 + 中间 P1 ticker + 右侧用户区”的结构。
  - `src/components/messages/P1Ticker.tsx`：抽成可嵌入顶栏的紧凑样式，保留关闭能力。
  - `src/components/layout/AppLayout.tsx`：去掉顶栏下方单独挂载的 `P1Ticker`，避免重复。

2. 修复左上角折叠侧栏按钮无效
- 当前按钮无反应的根因是：`AppSidebar` 使用了 `collapsible="none"`，而 `SidebarTrigger` 只对可折叠模式生效。
- 修复方案：
  - 将侧栏切换为真正可折叠模式；
  - 桌面端支持收起/展开；
  - 移动端保留抽屉行为。
- 同时保持你之前要求的轻量工作台风格，收起后不破坏布局。
- 对应实现会调整：
  - `src/components/layout/AppSidebar.tsx`
  - 视需要微调 `src/components/layout/AppLayout.tsx`
  - 不改动已有导航信息架构，只修正交互和收起后的展示方式。

3. 市场看板改成“顶部筛选 + 顶部摘要 + 下方全宽图表区”
- 取消当前市场看板左侧中栏承载筛选/摘要/公告的方式。
- 改为在右侧主操作区顶部顺序放置：
  1) 筛选区
  2) 行情摘要
  3) 规则预警（若保留）
  4) 所有图表模块全宽纵向排列
- 明确删除：
  - “公告信息”板块
- 行情摘要位置调整：
  - 放到筛选下面
  - 作为主工作区顶部摘要，不再放在独立中栏
- 图表区宽度调整：
  - “电价与价差”
  - “价差影响因子对照”
  - “负荷预测 vs 实际”
  - “新能源出力”
  - “竞价空间”
  - “市场运行与边界”
  全部改为主内容区全宽展示，顺序向下排，减少横向挤压。
- 这样页面只保留：
  - 左侧全局导航
  - 右侧单一主工作区
  不再有市场页自己的窄中栏。

4. 布局实现方式
- 优先直接复用现有 `WorkbenchLayout`，但让市场页不再提供左侧中栏内容；
- 如果现有壳层不适合单栏展示，则会在 `MarketInfo` 中改成更直接的单栏主区布局；
- 目标是避免再出现“左边一列筛选、右边一列图”的长期阅读割裂感。

5. 需要修改的主要文件
- `src/components/layout/AppTopBar.tsx`
- `src/components/messages/P1Ticker.tsx`
- `src/components/layout/AppLayout.tsx`
- `src/components/layout/AppSidebar.tsx`
- `src/pages/MarketInfo.tsx`
- 如有必要补充微调：
  - `src/components/layout/WorkbenchLayout.tsx`

6. 验收结果会对齐你的要求
- 顶栏不再显示“电力交易管理系统”和闪电。
- P1 消息进入顶栏同一行，且保留关闭 X。
- 左上角侧栏折叠按钮恢复可用。
- 市场看板筛选移动到主操作区最上方。
- 行情摘要放在筛选下面。
- 公告信息板块删除。
- 其余看板图表模块全部拉满主区宽度、顺序往下排，适合长时间连续查看。

技术说明
- 折叠按钮失效不是事件没绑上，而是侧栏当前被配置成“不可折叠”，所以触发器没有实际效果。
- 市场看板这次是布局重组，不是重做图表逻辑；现有图表、联动、tooltip、矩阵逻辑都可继续复用。
- 省份记忆逻辑已在 `ProvinceContext` 中存在，这次只调整它在市场页中的呈现位置，不改动记忆机制。
